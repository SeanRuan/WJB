# Branching Rules

## Branch names
- `feature/<short-topic>`
- `fix/<short-topic>`
- `docs/<short-topic>`
- `chore/<short-topic>`

## Rules
- Keep branch names short and clear.
- Use lowercase and hyphens.
- One branch = one task.
- Merge into `main` only after checks pass.

## Examples
- `feature/guild-members`
- `fix/readme-links`
- `docs/release-notes`
